# Interactive Security Dashboard - User Guide

## ЁЯОп Overview

The Interactive Security Dashboard is a modern, browser-based tool that visualizes and analyzes CSV reports generated by your PowerShell security audit scripts. It provides real-time charts, filtering capabilities, and comprehensive insights into your Azure and Office 365 security posture.

## ЁЯЪА Quick Start

1. **Open the Dashboard**
   - Simply open `dashboard.html` in your web browser
   - No installation or backend server required!

2. **Upload Your Reports**
   - Click the "Upload" tab
   - Drag and drop CSV files or click "Browse Files"
   - Upload multiple reports at once

3. **View Your Data**
   - Navigate through different security categories
   - Use filters to drill down into specific issues
   - View charts and statistics on the Overview tab

## ЁЯУК Dashboard Features

### Overview Tab
Shows aggregate security insights across all uploaded reports:
- **Total Reports**: Count of loaded CSV files
- **Critical Issues**: High-risk items requiring immediate attention
- **Warnings**: Medium-risk items for review
- **Average Score**: Overall security posture (0-100 scale)
- **Category Distribution Chart**: Pie chart of report categories
- **Risk Distribution Chart**: Bar chart showing risk levels
- **Security Scores Chart**: Category-wise security scores

### Category Tabs
Nine specialized views for different security areas:
1. **Azure IAM** - Identity and Access Management
2. **Data Protection** - Encryption and compliance
3. **Infrastructure** - Storage, network, and resource security
4. **Licensing** - Office 365 license management
5. **User Security** - User account and activity monitoring
6. **Email Security** - Exchange and mailbox security
7. **Teams Security** - Microsoft Teams configuration
8. **SharePoint** - SharePoint and OneDrive security

### Upload Tab
Manage your security reports:
- Drag and drop CSV files
- View all loaded reports
- Remove individual reports
- Clear all data at once

## ЁЯТб Usage Tips

### Best Practices
1. **Regular Updates**: Upload fresh reports weekly or monthly to track trends
2. **Focus on Critical**: Address critical issues before warnings
3. **Category Review**: Check each category systematically
4. **Filter Effectively**: Use filters to focus on specific risk levels
5. **Export Data**: Take screenshots of charts for documentation

### Workflow Recommendations
1. Run PowerShell audit scripts (see Scripts/ folders)
2. Upload generated CSV files to dashboard
3. Review Overview tab for high-level insights
4. Investigate categories with low scores or high risks
5. Use filters to identify specific issues
6. Take action on findings
7. Clear old data and re-upload updated reports

### Performance Tips
- Dashboard handles thousands of rows efficiently
- Load only necessary reports to save storage
- Clear old reports periodically
- Use filters instead of scrolling large tables

## ЁЯОи Understanding the Visualizations

### Category Distribution (Pie Chart)

Displays the count of items at each risk level:
- **Critical**: Immediate action required
- **High**: Priority attention needed

Shows average security scores per category (0-100 scale):
- **90-100**: Excellent security posture

- **50-74**: Moderate, several issues to address
- **Below 50**: Poor, requires immediate attention

## ЁЯФз Technical Details

### Browser Requirements
- Chrome 90+, Firefox 88+, Safari 14+, or Edge 90+
- JavaScript enabled
- localStorage enabled (not private/incognito mode)

### External Libraries (loaded via CDN)
- Chart.js 4.4 - Data visualization
- PapaParse 5.4 - CSV parsing
- Font Awesome 6.4 - Icons

### Data Format
CSV files should have:
- Header row with column names
- Standard columns recognized:
  - Risk columns: `RiskLevel`, `Risk`, `Status`
  - Score columns: `SecurityScore`, `Score`, `ComplianceScore`
  - Any additional columns from your scripts

### Storage Limits
- Browser localStorage: ~5-10MB
- Enough for dozens of reports
- Each report typically 10-500KB depending on size

## ЁЯРЫ Troubleshooting

### Reports Not Loading
**Issue**: CSV file uploaded but not showing in dashboard

**Solutions**:
1. Check filename contains recognized report type (e.g., "MFAStatus")
2. Verify CSV has proper header row
3. Check browser console (F12) for errors
4. Try refreshing page and re-uploading

### Charts Not Displaying
**Issue**: Charts are blank or not rendering

**Solutions**:
1. Ensure internet connection (for CDN libraries)
2. Refresh the page
3. Try a different browser
4. Clear browser cache

### Data Not Persisting
**Issue**: Reports disappear after closing browser

**Solutions**:
1. Ensure not using private/incognito mode
2. Check localStorage is enabled in browser settings
3. Verify sufficient storage space available
4. Try another browser

### Filter Not Working
**Issue**: Filters don't affect table data

**Solutions**:
1. Ensure column exists in the data
2. Check spelling matches exactly
3. Try clearing and reapplying filter
4. Refresh the page

### Slow Performance
**Issue**: Dashboard is slow with large datasets

**Solutions**:
1. Reduce number of loaded reports
2. Split large reports into smaller files
3. Use filters to narrow data view
4. Clear old reports you no longer need

```
1. Monday - Week 1: Run all Azure IAM scripts
2. Tuesday - Week 1: Upload IAM reports to dashboard, review findings
3. Wednesday - Week 1: Run Data Protection scripts
4. Thursday - Week 1: Upload and review data protection
```

```
1. Receive security alert
2. Run relevant audit script (e.g., External Forwarding)
3. Upload CSV to dashboard
4. Use filters to isolate suspicious activity
5. Cross-reference with other reports (e.g., User Security)
6. Document findings from dashboard visualizations
7. Take remediation action
8. Re-run script to verify fix
```

## ЁЯУИ Interpreting Results

### High Critical Count
- Review Infrastructure and IAM categories first

### Many Warnings
- Indicates areas needing attention
- Schedule regular maintenance
- May be opportunity for automation

### Uneven Category Distribution
- Could indicate gaps in monitoring
- Ensure all categories covered regularly
- Add missing audit scripts
